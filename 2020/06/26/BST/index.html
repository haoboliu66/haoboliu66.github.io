BST 有序表

经典BST中无重复值

1.BST的增删查

增和查操作相似

删除:

  1. 无left无right, 直接删

  2. 有left无right 或 有right无left(情况相同, 子树替他的位置),

  3. 有left有right, 找后继节点替他的位置(右树的最左节点), 后继节点如果有right child, 把right child交给后继节点的parent;

    用后继节点接替时要考虑后继节点是要删除节点的right child, 这种情况下, 后继节点无需交出自己的right child

裸的BST存在的问题:

输入数据状况决定性能: 数据问题导致树的不平衡, 复杂度无法保持在O(logN)

BST引入左旋和右旋构成平衡BST, 具备自平衡能力, 维持平衡的代价不超过O(logN)

常见平衡BST(self-balancing binary search tree)

AVL(Adelson-Velskii and Landis)

任何节点, 左右树高度差 < 2

Size-Balanced Tree

叔节点的个数不少于侄子节点的个数

Red Black Tree
