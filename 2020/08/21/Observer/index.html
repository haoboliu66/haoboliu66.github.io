Observer

Introduction

Observer design pattern(Publish/Subscribe) defines a one-to-many dependency between objects so that when one object changes state, all its dependents are notified and updated automatically.

UML

  1. Objects use Subject interface to register as observers or remove them from being observers.
  2. A subject can have multiple observers
  3. An observer provides an update interface to received(be invoked) signal from subject

Simple Example

Pros and Cons

优点

  • 降低了目标与观察者之间的耦合关系，两者之间是抽象耦合关系
  • 目标与观察者之间建立了一套触发机制
  • 支持广播通信
  • 符合“开闭原则”的要求

缺点

  • 当观察者对象很多时，通知的发布会花费很多时间，影响程序的效率
